HTTP协议是无状态的！同一个客户端的这次请求和上次请求是没有对应关系的。为了解决这个问题，web程序引入了cookie机制来维护状态。


HTTPS较HTTP安全

网络请求需要中间有很多的服务器路由器的转发。中间的节点可能篡改信息，而如果使用https，密钥只有你和终点才有。https利用ssl/tsl协议传输。它包含证书，缺载，流量转发，负载均衡，页面适配，浏览器适配，refer传递等，保证了传输过程的安全性

http协议与服务器交互的方法及区别

GET查       POST找     PUT增     DELETE删

区别

get提交数据会放在url上，以?分割url和传输数据，参数之间以&相连；post提交数据会放在HTTP包的body中

get数据大小有限制(url长度限制)，post无

get使用Request.QueryString获取变量值，post使用Request.Form获取变量值

post较get安全

HTTP状态码
1xx 提示成功 表示请求已被接收，继续处理
2xx 成功 表示请求已被接收，理解，接受
3xx 重定向 要完成请求必须进一步处理
4xx 客户端错误 请求语法错误或请求无法实现
5xx 服务端错误 服务端未能实现合法的请求
