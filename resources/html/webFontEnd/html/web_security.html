webWorker和webSocket

webWorker提供多线程，提高web应用计算效率。API：worker的主线程和子线程通过postMessage()来发送消息，通过向webWorker添加一个"onmessage"事件监听器来获取接收到的信息。

webSocket提高更高效的传输协议。webSocket:本质是一种协议。url前缀为ws://或wss://客户端喝服务端进行webSocket交互的方式可以理解为"HTTP握手+TCP数据传输"的方式。说它是HTTP握手：是因为浏览器喝服务器建立长连接的握手过程是按照HTTP1.1的协议发送的，有Request，Request Header，Response，Resopnse Header。说它是TCP传输：主要体现在建立长连接后，浏览器可以给服务器发送数据，服务器也可以给浏览器发送请求，它的数据格式并不是iji定义的，是在要传输的数据外层有ws协议规定的外层包的。
       webSocket的数据传输是以frame形式传输的，会将一条消息分为几个frame，按照先后顺序传输过去。
优点:
1.大数据的传输可以分为片段传输，不用考虑数据大小导致的 长度标志位不足的情况。
2.和http的chuck一样，可以边生成数据边传递消息，提高效率。
